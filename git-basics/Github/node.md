# What is Github?

**Github** is a *remote repo* — a tool that tracks changes to your files
over time so you can see what changed, when, and why over a remote repository. It's how nearly every
software team in the world collaborates on code. 

## Why use it?

- **History** — every saved change is recorded, so you can always go back.
- **Collaboration** — many people can work on the same project without
  overwriting each other.
- **Branches** — try out an idea without breaking the working version.

## The core idea

You take *snapshots* of your project, called **commits**. Each commit records
the exact state of your files at that moment, along with a short message
describing what you did:

```bash
git clone 'url'
git add .
git commit -m "Add the homepage layout"
```

Think of commits like save points in a game — you can always return to one.
